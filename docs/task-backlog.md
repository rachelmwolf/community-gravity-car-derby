# Task Backlog

## Purpose

This backlog converts the planner into issue-ready work. Each item should become a GitHub issue once ownership is assigned.

## Priority Legend

- `P0`: event cannot run correctly without this
- `P1`: high-value work that strongly affects quality and pacing
- `P2`: useful but should not displace critical execution work

## Governance And Event Definition

### P0

- Finalize gravity car derby format, race classes, and any limits on entries
- Finalize published rules and inspection standards
- Verify insurance, council coordination, and any venue or event approvals
- Confirm what “fairness” means operationally for this event and who makes final rulings

### P1

- Define success criteria for the pilot event
- Decide what gets documented live during the event for reuse next year

## Registration And Communications

### P0

- Define registration workflow and owner
- Prepare participant confirmation email
- Prepare participant reminder email with arrival instructions
- Define late-arrival and check-in-close policy
- Decide whether check-in windows should be staggered

### P1

- Prepare FAQ for common participant questions
- Prepare pre-event rule summary and inspection expectations
- Prepare post-event thank-you and follow-up communication
- Build a pre-race communication plan with timed reminders and resource prompts

## Volunteer Staffing

### P0

- Confirm Race Director
- Confirm Chief Inspector
- Confirm Safety Lead
- Confirm Event Host / MC
- Confirm Track Operations Lead
- Create role descriptions for every staffed function
- Build a volunteer signup form based on role fit and preference
- Identify backup coverage for every critical lead
- Confirm minimum staffing target for 60-car scenario

### P1

- Assign check-in lead and support
- Assign staging lead and finish / retrieval support
- Assign scoring / results support
- Assign setup / teardown crew
- Assign floater / runner coverage
- Assign hospitality / recruiting only after race-control staffing is secure

### P2

- Build a same-day reserve list for no-show coverage
- Add volunteer check-in sheet and contact list

## Race Operations

### P0

- Configure race software and test it before race day
- Prepare registration import or entrant data workflow
- Create inspection checklist
- Build the physical inspection station with the gauges and references needed for rule enforcement
- Define check-in flow
- Define pit / repair flow
- Define staging flow
- Define finals format and awards logic
- Define who can stop racing for safety or fairness reasons

### P1

- Prepare a printed race-day run-of-show
- Prepare an announcer script and ceremony notes
- Define dead-time filler plan between heats
- Decide whether appearance judging is separate from speed awards

### P2

- Build a heat pacing estimate for 40, 60, and 80 cars
- Build an incident log template for race-day use
- Choose award designs and identify who will 3D print them
- Choose or create check-in sizing-guide templates and identify who will print them

## Venue, Safety, And Logistics

### P0

- Finalize room or site layout
- Define participant flow from arrival to awards
- Confirm emergency access and first-aid location
- Prepare signage list
- Confirm inspection and pit table setup
- Confirm track setup requirements and teardown plan

### P1

- Review accessibility considerations
- Confirm parking and arrival guidance
- Confirm storage and security for equipment and awards

### P2

- Add printed venue map to the repo
- Add photo examples of successful setup

## Guest Experience

### P0

- Define opening ceremony flow
- Define awards ceremony flow
- Define who explains the event to first-time families

### P1

- Build hospitality or concessions plan
- Build recruiting table plan
- Build side-activity or downtime plan for families and siblings

### P2

- Create signage or scripts that reinforce the welcoming community tone

## Suggested Issue Launch Order

If these are turned into GitHub issues, open them in this order:

1. Finalize rules and inspection standards
2. Confirm critical leads
3. Define registration and participant communication workflow
4. Configure and test race software
5. Finalize check-in, inspection, pit, and staging flow
6. Finalize safety and venue plan
7. Create volunteer role descriptions and signup structure
8. Finalize awards and run-of-show
9. Fill flexible volunteer roles
10. Prepare printables, signage, and event-day materials
