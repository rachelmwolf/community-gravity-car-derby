# Build Rules Status

## Purpose

This document records what has already been decided about the build rules, what still needs confirmation, and what should be cleaned up in the source rules document for future organizers.

## Already Decided

These items were settled in committee discussion and are reflected in the repo planning docs.

### Maximum Height

- working decision: `4 inches` maximum height
- reason: expected to clear the MicroWizard K2 gate while staying simple for families

### Wheelbase

- wheelbase should not be treated as a separate pass / fail measurement
- simplified rule intent: wheelbase extensions are allowed as long as the car still fits the track and all published clearances

### Replacement Parts

- standard kit wheels and axles remain the baseline
- committee-approved same-type replacement wheels and axles are allowed
- performance parts such as bearings, bushings, washers, and reshaped wheels remain prohibited

### Failed Inspection Handling

- all cars may race
- only inspection-compliant cars are eligible for speed trophies
- all cars remain eligible for design recognition
- on-site fixes and reinspection are allowed when practical

## Still Open Or Not Fully Closed

These are the items that still need explicit follow-through.

### 1. Approved Replacement List

- the rules now allow committee-approved same-type replacement parts
- the committee still needs to publish the actual approved list for families
- this could be an Amazon list, written appendix, or both

### 2. Final Source-Doc Update

- the source `Gravity Car Derby Build Rules` document should be updated to match the decisions above
- until that happens, the repo and the source doc may drift

### 3. Inspection Language Cleanup

- the rules and inspection checklist should stay aligned on exactly what is checked quickly on race day
- if the source rules still contain hard-to-measure or confusing items, they should be simplified before publication

### 4. Support Tools Follow-Through

- the YouTube playlist is already linked in the source document
- the Amazon list is linked and can be updated without rewriting the source document
- replacement-part support materials can be tracked separately from the core rules text

## Cleanup Needed In Related Docs

Some repo docs were created before every rule decision was finalized, so future organizers should watch for stale wording.

### Race Operations

The race-operations guidance should match the final policy:

- compliant cars qualify for speed trophies
- non-compliant cars may still race
- non-compliant cars are ineligible for speed trophies
- non-compliant cars remain eligible for design recognition

Any older wording that suggests non-compliant cars cannot race should be treated as outdated.

### Staffing And Inspection Materials

Inspection-related docs should continue to reflect:

- the `4 inch` working height limit
- approved same-type replacement parts
- simplified wheelbase treatment
- design-award eligibility for all cars

## Recommended Future Resource

For reuse next year, keep three artifacts together:

1. the source build-rules document families receive
2. the short inspection checklist used at the table
3. this status log showing which rules were changed, why, and when

That will make it much easier for a future organizer to understand not just the final rules, but also the reasoning behind them.
