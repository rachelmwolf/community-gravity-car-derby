# Project Plan

## Purpose

This document converts the event into workstreams that can be tracked in GitHub issues and milestones.

## Core Workstreams

### Governance And Event Definition

- finalize event format
- finalize published rules
- verify insurance and council coordination needs
- confirm venue, access, and event boundaries

### Registration And Communications

- define registration process
- prepare participant communications
- prepare reminder cadence
- define arrival instructions and check-in windows

### Volunteer Staffing

- assign critical leads
- define volunteer-facing role descriptions
- recruit by role fit and preference
- create backup coverage plan
- deliver volunteer briefing

### Race Operations

- configure race software and timing
- define check-in, inspection, pit, and staging flow
- prepare awards process
- define race-day command structure

### Venue, Safety, And Logistics

- finalize room layout or site layout
- define signage package
- confirm first-aid plan and emergency path
- document accessibility considerations
- prepare setup and teardown plan

### Guest Experience

- opening ceremony and announcements
- hospitality and concessions
- recruiting and information area
- family-facing flow during race downtime

## Suggested First Milestones

- Foundation complete
- Registration and communications ready
- Volunteer model staffed
- Race-day operations validated
- Event-day ready
- Post-event retrospective
